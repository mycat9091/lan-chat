# 🌐 局域网聊天室 (LAN Chat)

一个基于 Node.js 和 Socket.io 的局域网实时通信工具，支持文本消息、文件传输和群聊功能。

## ✨ 功能特性

- 💬 **实时文本消息** - 支持多人实时聊天
- 📁 **文件传输** - 可在局域网内快速传输文件（最大 10MB）
- 👥 **群聊功能** - 支持多用户同时在线
- 🎨 **美观界面** - 现代化的 UI 设计
- 🔄 **实时更新** - 在线用户列表实时同步
- 📱 **响应式设计** - 适配不同屏幕尺寸
- 💾 **设备记忆** - 自动保存上次使用的昵称，快速进入
- 📜 **历史消息** - 保存最近 100 条消息记录，重新加入时自动同步

## 📋 系统要求

- Node.js 14.0 或更高版本
- 支持现代浏览器（Chrome, Firefox, Edge, Safari）

## 🚀 快速开始

### 1. 安装依赖

```bash
cd lan-chat
npm install
```

### 2. 启动服务器

**方式一：使用管理器（推荐）**
```bash
# 双击运行 manager.bat
# 或在命令行中运行
node manager.js
```

管理器提供以下功能：
- 启动/停止/重启服务器
- 查看服务器状态
- 查看本机 IP 地址
- 在浏览器中打开应用

**方式二：使用批处理脚本**
```bash
# 启动服务器
双击 start.bat

# 停止服务器
双击 stop.bat

# 重启服务器
双击 restart.bat
```

**方式三：使用 npm 命令**
```bash
npm start
```

服务器将在 `http://localhost:3000` 启动。

### 3. 访问应用

**本机访问：**
```
http://localhost:3000
```

**局域网其他设备访问：**
```
http://[服务器IP地址]:3000
```

> 💡 **提示**: 使用管理器可以方便地查看本机 IP 地址，其他设备使用该 IP 访问即可。

## 📖 使用说明

### 加入聊天
1. 在浏览器中打开应用
2. 输入您的昵称（系统会自动记住上次使用的昵称）
3. 点击"加入聊天"按钮
4. 自动加载最近 100 条历史消息

### 快速登录
- 如果之前登录过，输入框会自动填充上次使用的昵称
- 直接按 Enter 或点击"加入聊天"即可快速进入

### 清除记忆
- 如果想使用新昵称，点击"清除记忆"按钮
- 系统会清除保存的昵称信息

### 发送消息
- 在底部输入框输入消息，按 Enter 或点击"发送"按钮

### 传输文件
1. 点击左侧的 📎 图标
2. 选择要发送的文件（最大 10MB）
3. 文件将自动发送给所有在线用户
4. 接收方点击文件消息即可下载

### 查看在线用户
- 左侧边栏实时显示所有在线用户及人数

### 历史消息
- 重新加入聊天室时，自动加载最近 100 条历史消息
- 包括文本消息、文件传输和系统通知

## 🔧 技术栈

- **后端**: Node.js + Express + Socket.io
- **前端**: HTML5 + CSS3 + JavaScript (原生)
- **实时通信**: WebSocket (Socket.io)
- **文件处理**: Multer

## 🌐 获取本机 IP 地址

**推荐方式：使用管理器**
- 运行 `manager.bat`，选择"查看本机 IP 地址"

**手动查询：**


## ⚙️ 配置说明

### 修改端口
编辑 `server/index.js` 文件中的 `PORT` 变量：
```javascript
const PORT = 3000;  // 修改为您想要的端口
```

### 修改文件大小限制
编辑 `public/js/client.js` 文件：
```javascript
if (file.size > 10 * 1024 * 1024) {  // 当前限制 10MB
    alert('文件大小不能超过 10MB');
    return;
}
```

### 修改历史消息数量
编辑 `server/index.js` 文件：
```javascript
const MAX_HISTORY = 100;  // 当前保存 100 条历史消息
```

## 🔒 安全提示

- 此应用仅适用于**局域网内部使用**
- 不建议在公网环境下使用
- 文件传输未加密，请勿传输敏感信息
- 建议在可信的局域网环境中使用

## 🐛 常见问题

**Q: 其他设备无法连接？**
- 确保所有设备在同一局域网内
- 检查防火墙是否阻止了端口 3000
- 确认使用的是服务器的正确 IP 地址

**Q: 文件上传失败？**
- 检查文件大小是否超过 10MB
- 确保 `uploads` 目录存在且有写入权限

**Q: 消息发送不出去？**
- 检查网络连接
- 刷新页面重新加入聊天室
- 查看浏览器控制台是否有错误信息

**Q: 历史消息没有加载？**
- 确保服务器在运行中
- 刷新页面重新连接
- 历史消息仅保存在服务器内存中，重启服务器会清空

**Q: 如何清除保存的昵称？**
- 在登录界面点击"清除记忆"按钮
- 或手动清除浏览器的 localStorage
